MaxConcurrency

Gets the maximum number of concurrent identify messages that may be sent if the bot is sharded.

The logic behind this value is slightly complex. Given a maximum concurrency of 16 and 16 shards, all 16 shards may identify concurrently. However, given a maximum concurrency of 16 and 32 shards, the batching properties of this value come into play. Concurrently identifying shards 0 through 15 would be allowed, but not shards 0 through 8 and 16 through 24.

Effectively, only shards with different rate limit keys may concurrently identify, where the key is calculated as follows.

rate_limit_key = shard_id % max_concurrency
